Features

Features & tabs

Blossom is more than a single “start macro” button. Think of it as a small control room: each sidebar tab handles one part of your Sol's RNG session, and you turn on only what you need.

New here? Install and calibrate first on Getting Started, then come back to pick modules. Most features need at least one calibration before they work reliably.

How the app is laid out

When you open Blossom, you get a familiar desktop layout: sidebar on the left, a header bar with Start / Stop, and the active tab in the middle. Press Ctrl+K (or click the search box) to jump to any page or quick action without hunting through the sidebar.

Your session stats and biome info live in the side panel while the macro runs. Settings you change are saved right away — no “apply” button to forget.

Macro tab — the heart of it

This is where you enable automations. Blossom runs enabled modules in a fixed priority order each tick (buffs → merchant → Strange Controller → Biome Randomizer → biome selector → potions → daily quests → obby), so modules take turns instead of fighting each other.

Fishing vs everything else: While fishing runs, merchant, quests, potion craft, obby, BR/SC, and biome selector pause. If potion craft is also on, crafting wins priority and fishing waits — handy when you need brews between fish trips.

Merchant tab

Dedicated controls for teleporter shopping and Limbo return — not just toggles buried in Macro. You will see a three-step banner: Teleporter → Shop → Portable Crack.

  • Configure Mari / Jester / Rin buy lists and amounts
  • Enable teleporter route and Return to Limbo after merchant teleporter
  • Link to full walkthrough: Auto merchant guide

Webhook tab

Discord integration lives here. The Biome Notifier hub lists every biome with per-biome notify toggles, user/role IDs, and rare-biome ping modes. Merchant and aura webhooks use Blossom-styled embeds with your private server rejoin link.

Deep dive: Discord webhooks guide.

Auras tab

Aura detection thresholds, force-ping lists, and Discord user IDs for rare aura rolls. Works with log-based detection after you have calibrated. Webhook embeds show rarity, player, and optional screenshots.

Macro Calibrations

One hub for every click position and OCR region — merchant slots, fishing dock, reconnect button, potion UI, and more. Without this step, the macro is guessing. Calibration guide · Video tutorial

Appearance

Eight built-in themes (Pink, Dark, Light, OLED, Sakura, Midnight, Forest, Match system), accent override, UI scale, window size, reduce motion, and Custom UI CSS skins. Theming guide

Settings & extras

Misc toggles you will use often:

  • Private server link — reconnect and webhook rejoin fields
  • Mouse / input delay — raise if clicks land early (1000–2000 ms is a safe range)
  • Fishing failsafe rejoin — stuck at dock recovery
  • Runtime components — OpenCV and WinOCR sync status
  • Start/stop hotkeys, Roblox username for log matching

In-app changelog — always the freshest list of what changed in your build. Stable releases also post notes on GitHub Releases.

What to enable first

  1. Calibrate merchant + movements (minimum for most farms)
  2. Paste private server link if you want reconnect or webhook rejoin
  3. Turn on the modules you actually need — avoid enabling everything on day one
  4. Add Discord webhooks once biome/merchant detection works in-game
  5. Tweak Appearance when the macro side is stable

Why this matters: Blossom is built for overnight and long AFK sessions. Enabling only what you use keeps OCR load down, reduces mis-clicks, and makes troubleshooting easier when something drifts after a Roblox update.
