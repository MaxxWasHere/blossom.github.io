Webhooks

Discord webhooks

Webhooks let Blossom whisper to your Discord server when something worth your attention happens — a rare biome, Mari in shop, a godroll aura, or Eden spawning. You bring the webhook URL; Blossom formats the embed and optional pings.

Keep URLs private. Anyone with your webhook URL can post to that channel. Do not share screenshots that show the full URL. Blossom stores URLs in your local config only — they are never uploaded to GitHub or the license server.

Where to configure

Open the Webhook tab in Blossom. The Biome Notifier hub sits at the top — a table of every biome with notify toggles, Discord user IDs, role IDs, and rare-biome ping modes. Merchant and aura settings live on their respective tabs but use the same webhook URL fields in config.

Biome notifier

After biome detection is calibrated and running, Blossom reads Roblox logs for rolls. When a biome you enabled fires, it sends a Blossom-styled embed with biome info and your private server rejoin link so you can hop back in quickly.

  • Toggle notify per biome in the hub table
  • Add multiple user IDs or role IDs to ping (validation errors are clearer on current builds if IDs are malformed)
  • Rare biome mode — choose how aggressive pings are for high-tier biomes (see footnote under the table)
  • Aurora and Eggland were removed from the notifier list in recent releases

Your private server link in Misc settings fills the Rejoin field on biome and merchant embeds. Paste the same link you use for Auto Reconnect.

Merchant webhooks

Two moments can notify Discord:

EventWhat you get
Merchant found Embed when OCR detects Mari, Jester, or Rin — includes rejoin link; throttled so duplicate spam is reduced
Shop screenshot (optional) Full-screen capture after shop opens — toggle merchant_shop_webhook / UI equivalent

Per-merchant ping IDs can target different roles when each shop appears. Details: Auto merchant guide.

Aura detection (Auras tab)

Rare aura rolls use log-based detection with thresholds you set on the Auras tab:

  • Minimum rarity for webhook
  • Force-ping list for specific auras
  • Discord user ID to mention
  • Optional aura screenshot in the embed

Embeds use Blossom styling — rarity, player name, and artwork when available.

Eden contract

With Auto Eden Contract enabled on Macro, Blossom watches chat for the official “Eden has appeared” line (OCR + spoof guard, ~40 minute cooldown). On a real hit, it pings your webhook with a chat screenshot. Uses the same chat OCR calibrations as other features.

Currency & inventory screenshots

Periodical aura or inventory screenshots can run on timers when enabled on Macro. If Auto Potion Craft is on, currency screenshot webhooks are disabled and the currency calibration section hides — crafting and currency shots conflict.

Setup walkthrough

  1. In Discord: channel settings → Integrations → Webhooks → New Webhook → copy URL
  2. Paste URL into Blossom Webhook tab (or webhook_url in config)
  3. Set private server link in Misc for rejoin fields
  4. Enable biome detection + calibrate before expecting biome pings
  5. Test with a biome you can force or a debug roll — check channel permissions

Validation & privacy

  • User and role IDs must be numeric snowflakes — Blossom validates and shows readable errors
  • Webhooks send from your PC when events fire; no Blossom cloud relay
  • Do not commit config.json with webhook URLs to public repos

No pings at all

Confirm webhook URL, bot permissions in channel, and that biome detection or merchant OCR actually triggered in-game first.

Pings but no @mention

Check user/role ID spelling. Role pings need the role to be mentionable in that channel.

Duplicate merchant pings

Found webhooks throttle per merchant name (~45s). If OCR flaps between names, recalibrate OCR region.
