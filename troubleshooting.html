Troubleshooting

Most issues fall into a few buckets: the app shell, clicks/OCR, or beta licensing. Work through the section that matches what you see.

App won't start or window is blank

  • Install Microsoft WebView2 Runtime (Evergreen).
  • Run Blossom.exe or Blossom-beta.exe from releases — the bootstrap launcher, not only an old inner Blossom-x.y.z.exe unless you know you need it.
  • Try Run as administrator.
  • Watch the bootstrap splash for inline error text instead of a silent fail.
  • Full reinstall: delete %LOCALAPPDATA%\Blossom\ only if you have backed up config and calibrations.

Antivirus / "virus" warning

Blossom is open source. Windows Defender often flags PyInstaller exes heuristically. Upload the file to VirusTotal, compare hashes with GitHub releases, or run from source with py run_local_ui.py if you want to audit behavior directly.

Macro idle or clicks miss

  • Run Blossom as administrator.
  • Keep Roblox focused; use the same calibrated resolution and UI scale.
  • Increase mouse delay to 1000–2000 ms in settings.
  • Recalibrate OCR failsafe and shop regions — Calibrations.
  • Blossom no longer shrinks a maximized Roblox window when starting the macro — if focus fails, restore the window manually once.

Wrong potions / OCR misreads

  • Recalibrate merchant and OCR regions in Macro Calibrations.
  • Switch Roblox chat/UI font to Arial or Rubik.
  • Install Windows OCR language pack en-US for WinOCR.
  • Raise input delay; avoid translucent or scaled UI overlays on top of Roblox.

Fishing stuck or chat spam

  • Enable Fishing failsafe rejoin and Auto Reconnect in Misc — after 60s at dock with no bite, Blossom retries UI nav then reconnects.
  • Prep-phase stuck recovery runs after 90s without progress on collections/walking steps.
  • Chat close clicks are debounced — if chat blocks fishing, recalibrate dock path or use recorded fishing.json.

Reconnect fails

  • Set a valid Private Server link in Misc settings.
  • Calibrate Reconnect Start under Movements calibrations.
  • Ensure Auto Reconnect is enabled; check Roblox logs are readable (standard install path).

Runtime components (OpenCV / WinOCR)

  • Let the launcher finish syncing — progress shows on splash and in the runtime card.
  • WinOCR “missing” on a fresh install usually clears after the bundle downloads.
  • Fishing needs OpenCV; merchant/biome OCR needs WinOCR + en-US language pack.

Beta: license errors

  • Check internet, firewall, and that you are on the matching beta build.
  • /resethwid in Discord once if you changed PCs or see machine mismatch.
  • Beta keys are one device per key. Clearer timeout/SSL/DNS errors appear when the network is actually down.
  • If activation says it cannot reach the server from the .exe but dev mode works, update — packaged builds include HTTPS cert fallbacks.

Appearance / theming

  • Custom UI blocked? Remove @import, remote URLs, or oversize files — see Theming.
  • Settings not saving? Update to latest — appearance writes immediately to config on current builds.

Updates

Use the launcher Blossom.exe — it syncs to the newest release on your channel every start, including rollbacks. If sync fails, download manually from Releases and replace the launcher.

More help

Community Discord (link in repo README) or open a GitHub issue with Blossom version, Windows version, and what you already tried. Screenshots of calibration regions help for OCR issues.
