Theming

Theming

Make Blossom feel like yours. Pick a built-in palette in seconds, or drop in a single CSS file and reskin every surface in the app.

Two paths, one Appearance tab. Built-in themes swap colors, accents, and light/dark behavior. Custom UI loads your own .css from %LOCALAPPDATA%\Blossom\themes\ and overrides the whole look — sidebar, cards, modals, loading overlay, calibrations hub, and more.

Overview

Everything lives under Appearance in the sidebar. Blossom saves your choices to config immediately (theme, accent, Custom UI file, UI scale, window size, reduce motion).

Built-in themes

Eight palettes plus Match system. Fast, safe, and great if you only want a different mood — pink sakura, true-black OLED, mossy forest, and so on.

Custom UI

One CSS file restyles the existing app DOM. Override design tokens for a quick recolor, or target every documented selector for a full reskin.

Custom UI wins when active. Your built-in theme choice is kept as a fallback — set Custom UI to None to use it again. While Custom UI is on, the theme grid and accent swatches dim in the Appearance card.

JSON theme files from older builds are retired. Your .json files stay in the themes folder for reference; Blossom falls back to your saved built-in theme and shows a one-time notice.

Quick start — Custom UI in 5 steps

  1. Open Blossom → Appearance → click Open themes folder (or browse to %LOCALAPPDATA%\Blossom\themes\).
  2. Copy custom-ui.example.css to a new name, e.g. my-skin.css. Never edit the example file directly — Blossom may refresh it.
  3. Edit your copy: start with the §1 design tokens block for a fast recolor, or work through the selector sections for a full reskin.
  4. Back in Appearance → Custom UI, pick your file from the dropdown.
  5. Click Refresh. Your CSS applies immediately and persists across restarts.

Custom UI deep dive

Custom UI is CSS only. Blossom injects your stylesheet as a <style id="blsm-custom-ui"> tag. On startup, cached CSS applies before the first paint so you do not flash the default pink theme.

What Custom UI can do

  • Override every design token (--accent, --bg-card, radii, fonts, shadows).
  • Restyle any documented selector — sidebar, header, cards, modals, loading overlay, calibrations hub, webhooks table, runtime sync card.
  • Layout tweaks on the current DOM: flex order, grid, gaps, display:none (visual hide only), pseudo-element decorations.
  • Background images via data: URLs embedded in CSS (remote URLs are blocked).
  • Page-specific overrides with .page-content[data-blossom-page="Macro"] or html.blsm-page-macro-calibrations.

What Custom UI cannot do

  • Add or remove sidebar tabs, macro features, or settings sections.
  • Inject HTML, JavaScript, or React components.
  • Replace the app shell with a totally different UI structure — that requires a Blossom fork.

Files are sanitized on load. Unsafe CSS is rejected entirely (see Rules & security). Max file size is 512 KB. Only .css files in the themes folder are listed — not *.example.css.

The authoritative template is custom-ui.example.css (v3) in your themes folder. Blossom creates it on first Custom UI use. The sections below mirror every § block from that file.

Design tokens

Tokens are the “palette variables” Blossom reads everywhere — card backgrounds, accent pink, border colors. In Custom UI, edit the body { --accent: … } block first: many users never touch individual selectors because tokens alone reskin the whole app.

Rules & security

Blossom loads Custom UI through blossom_custom_ui.py. Files must pass sanitization or they are blocked with an error in Appearance.

RuleDetail
CSS onlyNo HTML, JavaScript, or executable content in theme files.
Max size512 KB per file.
FilenameMust end in .css, not .example.css, no .. paths.
No @importExternal stylesheets are blocked.
No remote url()http://, https://, ftp://, and protocol-relative URLs blocked. data: URLs are allowed.
No script vectors<script>, javascript:, vbscript:, expression(), behavior:, -moz-binding:, data:text/html blocked.
No embedded documents<iframe>, <object>, <embed>, <link>, <meta>, <base> blocked.

Example — full reskin starter

Start from the shipped template — it includes every § section with working defaults. Copy the whole file, then edit section by section.

/* my-skin.css — copy custom-ui.example.css, then customize */

body, body[data-theme="custom"] {
  --bg-root: #070b14;
  --accent: #818cf8;
  --accent-text: #e0e3ff;
  /* …all tokens… */
}

.sidebar {
  background: linear-gradient(180deg, #0a0f1d, #070b14);
  border-right: 1px solid var(--border);
}

.sidebar-item.active {
  background: var(--accent-glow);
  color: var(--accent-text);
}

.card {
  border-radius: var(--radius-lg);
  border: 1px solid var(--border-accent);
}

/* Target one tab only: */
.page-content[data-blossom-page="Macro"] .card {
  /* macro-specific tweaks */
}

For the complete starter, open %LOCALAPPDATA%\Blossom\themes\custom-ui.example.css — it is always kept in sync with the app (v3 marker).

Sharing themes

  1. Export your .css file (no personal paths or secrets — it is pure styling).
  2. Share on Discord, GitHub Gist, or a theme thread — include a screenshot if you can.
  3. Recipients drop the file into their themes folder, select it under Appearance → Custom UI → Refresh.

If your theme uses custom fonts, recipients must have those fonts installed locally — @font-face with remote URLs will not work. Prefer system stacks or embed via data: if you accept the file size cost.

Font licenses for Blossom’s bundled typefaces: see Credits.

Troubleshooting themes

ProblemFix
Theme not in dropdownFile must be .css, not .example.css, in the themes folder. Click Refresh.
“Blocked (unsafe CSS or too large)”Remove @import, remote url(), or script-like patterns. Keep under 512 KB.
Flash of default pink on startupShould not happen on current builds — CSS caches before first paint. Update Blossom if you see a flash.
Built-in theme picker grayed outExpected while Custom UI is active. Set Custom UI to None to use built-in themes again.
Changes not appearingSave the file, then click Refresh in Appearance. Restart Blossom if the bridge was still starting.
Light text on light backgroundSet html.blsm-light-ui overrides or use dark --text-primary when building a light skin.
JSON theme no longer worksJSON themes were retired in 2.2.x. Convert to Custom UI CSS using the token + selector reference above.
