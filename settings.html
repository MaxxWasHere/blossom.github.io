Settings & config

Almost everything you tune in Blossom lives in one AppData folder. You rarely need to edit JSON by hand — the UI writes changes immediately — but it helps to know where files go when you back up, migrate PCs, or share calibrations with a friend on the same resolution.

Where everything lives

%LOCALAPPDATA%\Blossom\
PathWhat's inside
app\stable\ or app\beta\Versioned Blossom payload — managed by Blossom.exe / Blossom-beta.exe; do not hand-edit
config.jsonMacro toggles, webhooks, appearance, timings — your main settings file
macro_calibs.json (or equivalent)Calibration coordinates — export/import from Calibrations hub
themes\Custom UI .css files + custom-ui.example.css
paths\Recorded walks: fishing.json, char_align.json, obby routes, etc.
runtime\Downloaded OpenCV / WinOCR bundles (hash-verified)
Potion .txt filesCraft recipes per biome — managed from Potion UI

Back up before reinstalling: Copy the whole Blossom folder (or at least config + calibrations + paths) if you wipe AppData.

Settings & extras page

The sidebar Settings & extras entry opens misc controls that do not fit on Macro:

  • Roblox username — matches log files for biome/aura detection
  • Private server link — Auto Reconnect deep link + webhook Rejoin fields
  • Mouse / input delay — milliseconds between UI actions; raise if mis-clicks
  • Start / Stop hotkeys — global hotkeys when configured
  • Fishing failsafe rejoin — dock stuck recovery (needs Auto Reconnect)
  • Runtime components card — OpenCV + WinOCR sync status and manual retry

Auto Reconnect

When Roblox crashes or disconnects mid-macro, Blossom can kill the client, open your private server link, and click the in-game Start button you calibrated. It verifies success from Roblox logs.

  1. Enable Auto Reconnect on Macro tab
  2. Paste Private server link in Misc (full Roblox share URL)
  3. Calibrate Reconnect Start under Movements in Macro Calibrations
  4. After reconnect during fishing, next cycle forces a sell if inventory was full

Appearance keys (UI)

Stored in the same config file — edited from Appearance tab:

  • ui_theme — pink, dark, light, oled, sakura, midnight, forest, system
  • ui_accent — optional accent override
  • custom_ui_css — filename in themes\ (empty = built-in only)
  • ui_scale, window width/height presets, reduce_motion

Full theming guide: Theming.

Stable vs beta & licensing

StableBeta
LauncherBlossom.exeBlossom-beta.exe
LicenseNone — freeDiscord key on first launch (/getkey)
Device binding—One key per machine; /resethwid in Discord if you change PC
UpdatesStable GitHub releases trackBeta releases track — separate payload folder

What you need to know about beta keys: Activation checks your key against the license service and binds it to your device. Keys are stored securely on your PC (not plain text). If activation fails after an update, check internet/firewall, ensure you are on the matching beta build, and use /resethwid once if you see a machine mismatch. No secrets or tokens are required on your side beyond the key Discord gives you.

Updater behavior

Keep one launcher shortcut forever. On every start, the bootstrapper syncs Blossom.exe (stable) or Blossom-beta.exe (beta) to the newest release on its channel — including deliberate rollbacks if the maintainer pulls a bad build. In-app update toasts offer download with progress bar; failed downloads show Retry.

Your config, calibrations, paths, and themes survive payload updates — only the inner versioned exe changes.

Import / export

  • Export calibrations from Macro Calibrations hub — share with friends on identical resolution/UI scale
  • Import config from the UI when migrating setups
  • Never share configs publicly with webhook URLs or private server links attached
