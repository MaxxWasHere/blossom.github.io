Getting Started

Getting Started

Install once, calibrate your screen positions, then flip on the automations you want. After that, Blossom mostly runs itself.

Install

  1. Download Blossom.exe from stable releases (or Blossom-beta.exe for beta).
  2. Run the launcher once. You will see a branded splash while it syncs the full app into %LOCALAPPDATA%\Blossom\app\{channel}\ and optional runtime components (OpenCV, WinOCR).
  3. Keep using the same launcher shortcut — it checks GitHub for updates every start, including deliberate rollbacks on your release track.

Beta builds show an activation screen on first launch. Get a key from the Discord bot with /getkey. One key per device.

First-time setup

  1. Calibrate — open Macro Calibrations in the sidebar. Follow Calibrations or watch the video tutorial. Without calibrations, OCR and clicks will miss.
  2. Misc settings — paste your Private Server link if you use Auto Reconnect or webhook rejoin fields.
  3. Macro tab — enable merchant, potions, fishing, anti-AFK, and anything else you need. Each module has its own toggles and timings.
  4. Webhook tab (optional) — add Discord webhook URLs for biome or merchant pings.
  5. Appearance (optional) — pick Pink, OLED, Sakura, or a Custom UI skin. See Theming.

Where files live

Settings, calibration JSON, potion files, walk paths, and custom themes all live under:

%LOCALAPPDATA%\Blossom\
Folder / filePurpose
app\stable\ or app\beta\Versioned Blossom payload (managed by launcher)
themes\Custom UI .css files and custom-ui.example.css
paths\Recorded walk paths (fishing.json, char_align.json, etc.)
Config JSONMacro toggles, webhook URLs, appearance — preserved across updates

Run the macro

Click Start in the header bar (or your configured hotkey). Blossom focuses Roblox and runs your enabled modules in a loop.

  • Run Blossom as administrator if clicks do not register in Roblox.
  • Keep the same resolution and UI scale you used when calibrating — fullscreen is recommended.
  • Starting the macro no longer shrinks a maximized Roblox window; focus restore only happens when the game was minimized.

Runtime components

Fishing vision needs OpenCV. Merchant, Eden, and biome-selector OCR use WinOCR (Windows OCR language pack en-US). Blossom downloads missing components on startup and shows progress in the runtime card — launch is not blocked while they sync.
