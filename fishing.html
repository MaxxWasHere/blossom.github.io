Fishing

Fishing mode

Fishing mode is the full dock loop from the original Noteab macro — brought into Blossom with vision-based bite detection, optional sell trips, and recovery when you get stuck. Turn it on when you want fish and currency without babysitting the rod.

Before you start: Enable Fishing Mode on the Macro tab, then calibrate fishing regions in Macro Calibrations. Without calibrations, Blossom cannot see the bite bar or click the reel reliably.

What happens each cycle

At a high level, Blossom repeats this loop while fishing is enabled:

  1. Prep — optional collections menu, character align, walk to dock (default path or your fishing.json recording)
  2. Cast & wait — watches for a bite (screen capture + bar color detection)
  3. Reel minigame — keeps the bar in the green zone by clicking at your calibrated position
  4. Optional sell — walks to fish shop, sells inventory, returns to fishing
  5. Repeat — other farm modules stay paused until fishing stops or yields to potion craft

First run note: The first time you use fishing on a fresh install, Blossom may download the OpenCV vision component (~47 MB saved from the main installer). It verifies the download by checksum and caches it under %LOCALAPPDATA%\Blossom\runtime. If download fails, a NumPy fallback still works — just slightly slower.

Calibrations you need

Open Macro Calibrations → Fishing. Mark each point while Roblox is fullscreen at the same resolution you will use for AFK. If a button moves after a game update, re-mark only that row.

Walk paths

Blossom ships default walks to the dock and seller. For awkward layouts or custom routes, record your own:

  • paths/fishing.json — dock approach and fishing prep replay
  • paths/fishing_sell.json — optional sell route
  • paths/char_align.json — keys-only alignment before paths play (no mouse jitter)

Record paths with tools/blossom_path_recorder.py in the source repo, or copy community presets that match your resolution. Set fishing_playback_multiplier in config if you need faster or slower replay.

Why paths matter: Roblox camera and collision differ per account setup. A recorded path that works on your PC beats a generic walk when you fish for hours and cannot afford getting stuck on a railing.

Sell trips & inventory

When sell-after-fishing is enabled, Blossom navigates to the fish shop, opens the sell tab, and clears your catch. After a successful Auto Reconnect, the next fishing cycle forces a sell first so a full inventory from a disconnect does not block new catches.

Failsafes — when you get stuck

Long AFK sessions hit edge cases. Blossom has recovery paths so you are not left in a menu overnight:

SituationWhat Blossom doesWhat you enable
60s at dock, no bite Retries in-game UI nav (backslash, S, A, Enter), then reconnects if still stuck Auto Reconnect + Fishing failsafe rejoin in Misc
90s stuck in prep (collections, walking) Same failsafe rejoin path Auto Reconnect + fishing failsafe
Chat window blocking clicks Debounced close-chat clicks; dock failsafe closes chat after UI nav Keep calibrations accurate; update to latest build

Reconnect setup: Private server link + Reconnect Start calibration under Movements.

Interaction with other modules

  • While fishing runs, merchant, daily quests, potion craft (unless it takes priority), obby, BR/SC, and biome selector pause.
  • Merchant teleporter and Limbo merchant toggles are ignored during fishing mode.
  • Biome detection and webhooks can still run from logs in the background.

Troubleshooting fishing

Reel minigame misses the bar

Recalibrate bar region and click position. Raise mouse delay if clicks fire before the UI settles.

Never detects bites

Check bite pixel calibration and that OpenCV installed (runtime card on Settings). NumPy fallback works but needs accurate regions.

Stuck at dock overnight

Enable failsafe rejoin + Auto Reconnect. Verify private server link and Reconnect Start calibration.

Sell trip fails

Re-mark shop buttons. Consider a custom fishing_sell.json path if the default walk does not match your layout.
